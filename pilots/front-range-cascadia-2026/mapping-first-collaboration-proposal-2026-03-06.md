# Mapping First: A Shared Practice for Bioregional Knowledge Commoning

## Thesis

The first shared practice for inter-bioregional knowledge commoning should be mapping.

Not mapping only as cartography, but mapping as shared semantic orientation:

- who is here
- what is happening
- what needs attention
- what practices already exist
- what indicators matter
- what should remain local or consent-gated

If that layer is weak, later work on funding, matching, commitments, and protocol reuse becomes premature.

## Why the Map Is Not the Model

A geographic map is useful, but it should not become the canonical model.

The canonical model should be a graph-native semantic layer that can render into multiple views:

- geographic view for place and watershed orientation
- graph view for relationships and learning loops
- table view for operational coordination
- narrative view for knowledge-garden and public communication

This keeps the same underlying knowledge reusable across analysis, governance, interfaces, and AI systems.

## Why Mapping Comes Before Funding

Funding mechanisms need legible context.

Before capital can move well, communities need enough shared structure to answer:

- What is the problem?
- Where is it located?
- Who is already stewarding it?
- What practices are already underway?
- What evidence exists?
- What indicators would show change?
- What authority and consent boundaries apply?

Without that, funding systems risk optimizing proposal theater instead of real bioregional learning.

## What Mapping Produces

A useful mapping packet should be able to represent:

- one bioregion or place
- the organizations active there
- the practices they are running
- the projects coordinating those practices
- the key questions and claims in play
- the evidence and indicators attached to those claims
- the relationships between local terms and shared bridge concepts

## How This Supports Ontological Commoning

Mapping should not force everyone into one ontology.

Instead:

- local and domain-specific concepts remain visible
- bridge mappings are proposed and reviewed
- repeated local concepts can become extension candidates
- cross-bioregion comparability grows from use, not imposition

This allows plurality while still supporting machine-readable coordination.

## How Practices Become Patterns Without Flattening Difference

Pattern mining should not start from abstraction alone.

It should start from documented practices in their local context, then compare across places. A practice only becomes a pattern candidate after enough grounded comparison shows that the abstraction is actually helpful.

That means the flow is:

`local practice -> mapped artifact -> comparison -> pattern candidate -> review -> protocol or pattern reuse`

This preserves local texture while still enabling trans-local learning.

## Why This Matters for Cascadia and Front Range

Cascadia and Front Range do not need to have the same ecological conditions or political context to learn from each other.

What they do need is a shared way to compare:

- practices
- evidence
- questions
- governance approaches
- indicators
- ontology differences

A graph-first mapping layer makes that comparison much easier than isolated notes, spreadsheets, or flat maps.

## Suggested First Shared Sprint Output

A first shared sprint should aim to produce three small things, not one giant platform:

1. One Cascadia mapping packet.
2. One Front Range mapping packet.
3. One comparison note identifying:
   - shared pattern candidates
   - local differences that should remain local
   - ontology terms that need review or extension

## Async Contribution Path

Someone does not need to adopt the full BKC stack to participate.

A useful async contribution could be:

- one place packet
- one organization packet
- one practice packet
- one interview packet with consent metadata
- one indicator packet tied to a location or question

Those can be translated into the shared bridge layer while preserving source ontology and local meaning.

## What This Intentionally Does Not Require

This proposal does not require:

- a final shared worldview ontology
- a final Boulder boundary decision
- a full GIS stack before knowledge work starts
- a funding mechanism before mapping starts
- a universal agreement on all categories

It only requires enough shared semantic structure to compare, learn, and extend responsibly.
